# FO Test Harness — Agent Instructions

This repo contains the FounderOps BUILD → QA → DEPLOY harness plus intake tooling.
Use this file as the base operating guide for any agent working in this codebase.

## What This Repo Does
- Converts founder answers into structured intake JSON.
- Runs a Claude BUILD + ChatGPT QA loop to produce artifacts.
- Packages results into a ZIP under `fo_harness_runs/`.

## Key Entry Points
- `fo_test_harness.py`: Main orchestration pipeline (BUILD → QA → optional DEPLOY).
- `intake/run_intake_v7.sh`: Generates intake JSON from hero answers.
- `intake/convert_hero_answers.py`: Converts raw founder text into hero JSON.
- `check_boilerplate_fit.py`: Analyzes whether a startup fits the boilerplate.

## Common Workflows
1. Intake generation:
   - `cd intake`
   - `./generate_intake.sh hero_text/<startup>.json`

2. Build + QA loop:
   - `python fo_test_harness.py --intake <intake.json> --startup-id <id> --block B --build-gov <zip> --tech-stack <custom|lowcode>`

3. Review outputs:
   - `fo_harness_runs/<startup>_BLOCK_<X>_<timestamp>/`

## Safety & Repo Hygiene
- Never commit or expose API keys. Keys should only be in environment variables.
- Treat `.claude/settings.local.json` as sensitive; do not copy keys from it.
- Avoid modifying `fo_harness_runs/` and `boilerplate_checks/` unless explicitly asked.
- Preserve build artifacts and QA logs when changing the harness.
- Directive: Do not write or modify code unless you receive an explicit “YES” from the user.
- Always commit `fo_run_log.csv` when it changes.
- Outside of actual builds, default to ChatGPT instead of Claude to control costs.

## Notes for Code Changes

- `build_state.json` and `artifact_manifest.json` are generated by the harness.
