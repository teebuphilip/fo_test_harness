**BOILERPLATE INTEGRATION MODE - MANDATORY RULES**
**═══════════════════════════════════════════════════════════════**

**CRITICAL: You are building BUSINESS LOGIC ONLY, not a complete application.**

**BOILERPLATE ASSUMPTION:**
The teebu-saas-boilerplate already provides:
- ✅ Authentication system (user signup, login, sessions)
- ✅ Payment processing (Stripe integration, subscription management)
- ✅ Database ORM (connection, migrations, base models)
- ✅ Email system (transactional emails, templates)
- ✅ UI framework (React/Vue components, CSS framework, layouts)
- ✅ API infrastructure (Express routes, middleware, error handling)
- ✅ Deployment scripts (Docker, environment config, CI/CD)

**YOUR JOB:**
Build ONLY the business-specific modules that extend the boilerplate:

**RULE 1: Business Models Only**
Create domain models that represent business entities:
- ✅ DO: Create Horse.js, Race.js, BreedingRecord.js models
- ❌ DON'T: Create User.js, Payment.js, Session.js models (boilerplate has these)

**RULE 2: Business Services Only**
Create services for domain-specific logic:
- ✅ DO: RacingDataService.js, HorsePerformanceAnalyzer.js
- ❌ DON'T: AuthService.js, PaymentService.js, EmailService.js (boilerplate has these)

**RULE 3: Business UI Components Only**
Create UI components specific to your domain:
- ✅ DO: HorseProfileCard.jsx, RaceResultsTable.jsx
- ❌ DON'T: LoginForm.jsx, PaymentForm.jsx, Navbar.jsx (boilerplate has these)

**RULE 4: Configuration Over Infrastructure**
Provide config files that tell the boilerplate how to behave:
- ✅ DO: wynwood-config.js with membership tiers, pricing, features
- ❌ DON'T: Rebuild Stripe integration, auth flows, or database setup

**RULE 5: Integration Hooks**
Your code should integrate with boilerplate hooks:
- ✅ DO: Export functions that boilerplate can call
- ✅ DO: Import from boilerplate base classes
- ❌ DON'T: Create parallel systems that duplicate boilerplate features

**FILE: config/wynwood-config.js**
```javascript
// Configuration for boilerplate
module.exports = {
  businessName: 'Wynwood Thoroughbreds',

  // Tell boilerplate what membership tiers to create
  membershipTiers: [
    { id: 'paddock', name: 'Paddock Pass', price: 500 },
    { id: 'winners', name: "Winner's Circle", price: 1200 },
    { id: 'elite', name: 'Elite Syndicate', price: 2000 }
  ],

  // Tell boilerplate what features each tier gets
  tierFeatures: {
    paddock: ['horse_updates', 'education_library'],
    winners: ['horse_updates', 'education_library', 'founder_messages', 'events'],
    elite: ['horse_updates', 'education_library', 'founder_messages', 'events', 'consultations']
  },

  // Custom routes to mount
  customRoutes: [
    { path: '/horses', handler: require('./routes/horses') },
    { path: '/races', handler: require('./routes/races') }
  ]
};
```

**FILE: models/Horse.js**
```javascript
// Extend boilerplate's BaseModel
const { BaseModel } = require('@teebu-saas/models');

class Horse extends BaseModel {
  static tableName = 'horses';

  static schema = {
    name: 'string',
    breed: 'string',
    age: 'integer',
    performance_data: 'json'
  };

  // Business-specific method
  async getPerformanceAnalysis() {
    const RacingDataService = require('../services/RacingDataService');
    return RacingDataService.analyzeHorse(this.id);
  }
}

module.exports = Horse;
```

**FILE: services/RacingDataService.js**
```javascript
// Business logic service
const { ApiClient } = require('@teebu-saas/integrations');

class RacingDataService {
  constructor() {
    this.drfApi = new ApiClient({
      baseUrl: 'https://api.dailyracingform.com',
      apiKey: process.env.DRF_API_KEY
    });
  }

  async fetchHorseData(horseId) {
    // Business-specific external integration
    return this.drfApi.get(`/horses/${horseId}`);
  }

  async analyzeHorse(horseId) {
    // Business logic
    const data = await this.fetchHorseData(horseId);
    return this.calculatePerformanceScore(data);
  }
}

module.exports = RacingDataService;
```
